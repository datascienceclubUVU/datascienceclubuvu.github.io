BEGINNER SQL: HELPFUL KEYWORDS



Introduction

Writing queries in SQL can either be very simple or very complex. Rarely are they ever in between. To help you expand your SQL tool belt, this tutorial will introduce you to some common keywords used in conjunction with the 5 basic SQL statements we learned in the first tutorial.

Helpful Keywords

In the previous tutorial, we looked at table and column constraints and the benefits they provide performance-wise. In addition to those constraints, SQL also provides an array of keywords to be used in the queries themselves. Listed below are some common keywords and their use cases:

  • DISTINCT - This keyword is used to eliminate duplicate rows. This is very useful for identifying the actual number of cutomers and
               reducing errors in your analyses.
       Example: SELECT DISTINCT customer_id FROM customers;
  • TOP - This keyword is used to limit the number of rows returned. This is very useful when working with massive datasets.
       Example: SELECT TOP 100 * FROM customers;
  • LIKE - This keyword is used in conjunction with the WHERE clause. This is used to do a "fuzzy" match with strings that contain certain
           characters.
       Example: SELECT * FROM customers WHERE firstName LIKE 'Mic%';
  • IN - This keyword is also used in conjunction with the WHERE clause. This is used to identify items in a list.
       Example: SELECT * FROM customers WHERE firstName IN('Aaron', 'Carlos', 'Stacy');
  • OFFSET - This keyword is used in conjunction with the ORDER BY clause. This is used to skip "n" rows from the top of the result set.
       Example: SELECT * FROM customers ORDER BY firstName OFFSET 10 ROWS;
