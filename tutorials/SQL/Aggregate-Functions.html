SQL Tutorials: Aggregate Functions

DISCLAIMER: ANSI SQL (The SQL that's used universally) includes everything taught in this section. However, some of the syntax shown will vary between your Relational Database Management Service (RDBMS).

Isn't querying fun? Data Analysis is all about asking and answering questions using querying techniques such as SELECT statements. Let's expand our querying skills by learning how to work with numerical data. This lesson will cover the following learning objectives:

  • Arithmetic Operators
  • COUNT Function
  • SUM Function
  • AVERAGE/AVG Function
  • MIN/MAX Functions

Arithmetic Operators



Summary:

  • If you want to add two columns together, add a column by a certain value, or concatenate two or more columns, use the + symbol with the following syntax:
    SELECT [column1] + [column2]
    FROM [table_name];
  • If you want to subtract one column from another or subtract the values in a column by a certain amount, use the - symbol with the following syntax:
    SELECT [column1] - [column2]
    FROM [table_name];
  • If you want to multiply one column by another or multiply the values in a column by a certain amount, use the * symbol with the following syntax:
    SELECT [column1] * [column2]
    FROM [table_name];
  • If you want to divide one column from another or divide the values in a column by a certain amount, use the / symbol with the following syntax:
    SELECT [column1] / [column2]
    FROM [table_name];
  • If you are performing a division task and want to get the remainder, use the % symbol with the following syntax:
    SELECT [column1] % [column2]
    FROM [table_name];
  • NOTE: Although SQL isn't a Programming Language by nature, you can still declare variables and perform arithmetic operations on them. We will discuss these in a leter tutorial.
  • NOTE: When creating new columns from arithmetic operators, it's smart to give the new column an alias. This can be done using the AS keyword with the following syntax:
    SELECT [column1] + [column2] AS [new_column_name]
    FROM [table_name];

COUNT Function

The COUNT Function can be used for both finding the number of items in a group or finding the total number of rows in a table, as seen in the previous tutorial. Watch the following video to get a better understanding of this function:

    Intro to SQL Count Function

Summary:

  • To retrieve the total number of rows in a table, use the following syntax:
    SELECT COUNT(*)
    FROM [table_name];
  • To retrieve the total number of rows in a filtered result set, use the following syntax:
    SELECT COUNT(*)
    FROM [table_name]
    WHERE [condition(s)];
  • To retrieve the total number of DISTINCT items in a specified column, use the COUNT function with the DISTINCT keyword using the following syntax:
    SELECT COUNT(DISTINCT [col_name])
    FROM [table_name];
  • To retrieve the total number of NON-NULL rows in a specific column, use the following syntax:
    SELECT COUNT([col_name])
    FROM [table_name];

SUM Function

Gathering totals is probably the most common type of analytics activity. In SQL, this is conducted using the SUM function. It should be noted that this function only works on NUMERIC columns. Watch the following video to get a better understanding of this function:

    Intro to SQL Sum Function

Summary:

  • To retrieve the total of all values in a specific column, use the following syntax::
    SELECT SUM([col_name])
    FROM [table_name]
  • To retrieve the total of all values in a specific column based on a set of conditions, use the following syntax:
    SELECT SUM([col_name])
    FROM [table_name]
    WHERE [col_name] = [value];
  • When working with aggregated columns, it's best to rename the columns. This can be done with the following syntax:
    SELECT SUM([col_name]) [new_name]
    FROM [table_name];
  • To calculate the average of one or more columns, use the following syntax:
    SELECT SUM([col_name]), COUNT([col_name]), SUM([col_name])/COUNT([col_name])
    FROM [table_name];

AVERAGE/AVG Function

If you've taken basic Algebra, you'll know that an average of a set of numbers is computed as the sum of that list divided by the count of the items in that list. However, SQL enables you to reduce this to one step using the AVERAGE/AVG function (the actual function name differs between RDBMSs). Watch the following video to get a better understanding of this function:

    Intro to SQL Average Function

Summary:

  • To retrieve the average value from a specific column, use the following syntax (differs between RDBMSs):
    SELECT AVG([col_name])
    FROM [table_name];
  • To retrieve the average value from a specific column based on a set of conditions, use the following syntax (differs between RDBMSs):
    SELECT AVG([col_name])
    FROM [table_name]
    WHERE [col_name] = [value];
  • QUICK NOTE: Some RDBMSs may include NULL row values when calculating averages. In this scenario, use the following syntax to calculate the average:
    SELECT SUM([col_name])/COUNT([col_name])
    FROM [table_name];

MIN/MAX Functions

When doing Data Analysis, it's good to know the range of values you're working with. This can be found using SQL's MIN and MAX Functions. Watch the following video to get a better understanding of these functions:

    Intro to SQL Min Function
    Intro to SQL Max Function

Summary:

  • To retrieve the smallest number in a specific column, use the following syntax:
    SELECT MIN([col_name])
    FROM [table_name];
  • To retrieve the smallest number in a specific column based on a set of conditions, use the following syntax:
    SELECT MIN([col_name])
    FROM [table_name]
    WHERE [col_name] = [value];
  • To retrieve the largest number in a specific column, use the following syntax:
    SELECT MAX([col_name])
    FROM [table_name];
  • To retrieve the largest number in a specific column based on a set of conditions, use the following syntax:
    SELECT MAX(col_name])
    FROM [table_name]
    WHERE [col_name] = [value];
