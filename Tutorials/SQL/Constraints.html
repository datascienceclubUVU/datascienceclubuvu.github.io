BEGINNER SQL: CONSTRAINTS



Introduction

Databases are powerful tools that can drive business value and prevent data loss. However, database objects can't determine when a logical error (such as whether or not you ment to delete data) occurs. To help with this, RDBMSs allow users to add rules, known as CONSTRAINTS to prevent mistakes and optimize query performance. In locally based RDBMSs, as opposed to cloud-based RDBSms, there are two types of object constraints: Table Constraints and Column Constraints.

TABLE CONSTRAINTS

TABLE CONSTRAINTS allow you to place rules or limits on a database table to reduce errors, keep data consistent, and optimzie query performance. There are three primary types of table constraints:

  • PRIMARY KEY Constraint - This specifies the primary key of the table.
  • FOREIGN KEY Constraint - This specifies the primary key and the table to reference. This column must be the same data type.
  • INDEX Constraint - This specifies the column in which the index is based on. An INDEX is an external tool used to quickly look up rows in a large query.

These constraints enable you to maintain order within your database objects and reduce the likelihood of redundancy and data quality errors. Indexes are very useful when your table is in excess of 10,000 rows. These will be discussed in a later tutorial.

Example

   CREATE TABLE salesReports (reportID INT PRIMARY KEY, totalSales INT, foodSales INT,
   cleaningSales INT, clothingSales INT, month VARCHAR(25), year INT,
   storeID INT FOREIGN KEY REFERENCES stores (storeID);)

   CREATE INDEX idx_storeID ON salesReports (reportID);


COLUMN CONSTRAINTS

COLUMN CONSTRAINTS allow you to place limits on the data inserted into your tables, rules for how data is presented, and compress certain data types to save space. There are four primary types of column constraints:

  • NOT NULL Constraint - This forces the user to enter a value for the specified column.
  • IDENTITY - Primarily used for primary keys, this auto-increments the column's values. This is only allowed for integer columns.
  • CHECK Constraint - This limits the data that can be inserted into a column value to a list of options.
  • UNIQUE Constraint - This forces the user to enter only unique valeus for each row in the column.
  • DEFAULT Constraint - If there is not a NOT NULL constraint, this value sets a default value for any NULL values.

Example

   CREATE TABLE salesReports (reportID INT PRIMARY KEY IDENTITY(1, 1), totalSales INT NOT NULL,
   foodSales INT DEFAULT 0, cleaningSales INT, clothingSales INT,
   month VARCHAR(25) CHECK (month IN('January', 'February', 'March', 'April', 'May', 'June',
   'July', 'August', 'September', 'October', 'November', 'December')), year INT,
   storeID INT FOREIGN KEY REFERENCES stores (storeID);)
