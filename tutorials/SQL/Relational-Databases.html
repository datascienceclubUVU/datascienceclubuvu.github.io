Database Objects

To keep things organized, Relational Database Management Systems (RDBMSs) create business logic in a hierarchy of objects. These include Databases, Schemas, Tables, and Attributes. view the videos below to learn more about these objects:

    What is a Relational Database?
    Why Are Tables Used in Relational Databases?
    What is a Database Schema?

Summary:

  • A Relational Database is a system comprised of tables that represent real-world objects and contain Attributes (Columns) and rows of data.
  • A Table is a unit of data that contains Columns (Attributes) and Rows (Records) to describe the data.
  • A Schema is a collection of tables that have a common structure and have limitations called constraints to ensure data quality.

Primary Keys vs. Foreign Keys

The main reason data professionals prefer relational databases over spreadsheets is their relational properties that come in the form of primary keys and foreign keys. Take a look at the videos below to learn about this concept:

    Primary Keys vs. Foreign Keys Tutorial

Summary:

  • Primary Keys are the central component of relational databases and are used to uniquely identify rows in a table.
  • Foreign Keys are primary keys stored in another table to create a relationship between two tables.
  • Composite Keys are primary keys composed of more than one column. These are used when more than column is requried to uniquesly identify each row in a table.
  • Candidate Keys are columns that can be used to uniquely identify each row in a table, though haven't been designated as primary keys yet.
