Definitions

Before we dive into the semantics of SQL, let's take a look at some common terminology:

  • Column: Also called an "attribute" or "field", this describes an aspect of a data point.
       Example: Height, weight, and age may all be columns.
  • Row: Also called a record, this is a collection of columns that represent a single data point.
       Example: One row may represent one hospital patient and may have attributes like age, weight, and height.
  • Table: This is a collection of rows and columns. This is the most basic unit found in a database and will be the foundation of this tutorial.
  • Query: This is a specific selection of columns from a table that can include filters to limit the rows retrieved. A google search is an example of a query.

The 5 Basic SQL Statements

There are five components to a basic query:

  • SELECT (mandatory): This is how to start a query and is used to specify the columns to return.
  • FROM (mandatory): This is used to specify the table to query.
  • WHERE: This is used to filter the rows that are returned.
  • GROUP BY: This is used in conjunction with aggregate functions (SUM, COUNT, MAX, etc.) to specify the level of detail to provide.
  • ORDER BY: This is used to specify the sort order of the results.

The structure is as follows:
   SELECT [column1], [column2], [column3]
   FROM [table name]
   WHERE [condition]
   GROUP BY [all columns NOT being aggregated]
   ORDER BY [column1], [column2] ('ASC' (default)/'DESC')

EXAMPLE

   SELECT customerID, name, age, SUM(numOrders)
   FROM customers
   WHERE age > 20
   GROUP BY customerID, name, age
   ORDER BY customerID;
