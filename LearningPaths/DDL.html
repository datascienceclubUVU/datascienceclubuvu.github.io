BEGINNER SQL: Data Definition Language (DDL)



Introduction

Databases are a collection of objects including tables, views, and stored procedures. All of these objects don't just magically appear as you need them, they must be created using a subset of SQL known as Data Definitiona Language (DDL). Let's dive right into this sub-language.

DDL Statements

DDL offers a wide array of statements to create, alter, and drop database objects. Listed below are the most common DDL statements used when creating and maintaining a Relational Database Management System (RDBMS):

  • CREATE DATABASE [database name]: This allows you to create a database.

  • CREATE SCHEMA [schema name]: This allows you to create a schema within the database in which you are running the statement. A schema is a database object that is a collection of tables.

  • CREATE TABLE [schema name].[table name] ([column name1] [data type], [column name2] [data type], [constraint1]): This allows you to create a table and specify its columns and their associated data types and specify its schema within the database in which you are running the statement.

  • CREATE VIEW [view name] AS ([query]): This allows you to create a view, a custom table created from a SELECT query, within the database in which you are running the statement.

  • ALTER TABLE [schema name].[table name] ADD [column name] [data type]: This allows you to add a new column to the specified table and specify its data type.

  • ALTER TABLE [schema name].[table name] ALTER COLUMN [column name] [new data type]: This allows you to change the data type of the specified column in the specified table.

  • DROP TABLE [schema name].[table name]: This allows you to remove the specified table from the database in which you are running the statement.

  • DROP VIEW [view name]: This allows you to remove the specified view from the database in which you are running the statement.

  • DROP SCHEMA [schema name]: This allows you to remove the specified schemas and the tables it contains from the database in which you are running the statement.

  • DROP DATABASE [database name]: This allows you to remove the specified database and the schemas, tables, and views it contains from the server.

Example

   CREATE DATABASE sales;
   CREATE SCHEMA managers;
   CREATE TABLE managers.salesReps (repID INT PRIMARY KEY, firstName VARCHAR(25), lastName(25), hireDate DATE);
   CREATE VIEW totalSalesByRep AS ( SELECT repID, firstName, lastName, hireDate, FROM salesReps
                                    WHERE hireDate >= '01-01-2017' AND < '01-01-2019');
