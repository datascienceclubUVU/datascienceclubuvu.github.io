SQL Tutorials: Intro to SELECT Statements

DISCLAIMER: Before starting this tutorial, it's highly recommended that you download an RDBMS to follow along. We recommend using either PostgreSQL, MS SQL Server, or MySQL.

Hopefully the lessons in this learning series are giving you a good understanding of the underlying nature of relational databases because from here on out, we will be working almost exclusively with SELECT statements. SELECT statements are the most common statements in SQL and are sued to execute queries on tables and views (we cover those in a later tutorial). To get started with SELECT statements, we'll cover the following objectives:

  • SELECT Statement Syntax
  • WHERE Clause
  • GROUP BY Clause
  • ORDER BY Clause

SELECT Statement Syntax

As mentioned in the introduction, the SELECT statement is the most common statement in SQL, therefore it's critical to master the basic syntax. Watch the following video to get a better understanding of SELECT statement syntax:

    SELECT Statement Basics

Summary:

  • The Result Set is the rows returned as a result of the query executed.
  • To retrieve all rows in a specified table, use the following syntax:
    SELECT *
    FROM [table_name];
  • To retrieve the total number of rows in a specified table, use the following syntax (We will go over the COUNT function in the next tutorial):
    SELECT COUNT(*)
    FROM [table_name];
  • To retrieve all the rows in a specified table but limit which columns are returned, use the following syntax:
    SELECT [col1], [col2], [col3], [col4]
    FROM [table_name];
  • To retrieve all rows in a specified table that match a specified condition, use the WHERE clause with the following syntax:
    SELECT *
    FROM [table_name]
    WHERE [condition(s)];
  • To order the result set in either ascending or descending order, use the ORDER BY clause with the following syntax:
    SELECT *
    FROM [table_name]
    WHERE [condition(s)]
    ORDER BY [col_name] [ASC/DESC];
  • When working with large result sets, it's a best practice to use the LIMIT Keyword to reduce the amount of memory your query uses:
    SELECT *
    FROM [table_name]
    WHERE [condition(s)]
    ORDER BY [col_name] [ASC/DESC]
    LIMIT [integer];

WHERE Clause

Although you have used the WHERE clause before, there's a lot more to it. Let's explore this clause in the video below:

    How to Filter with the WHERE Clause in SQL

Summary:

  • To filter the result set with a single condition, use the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] = [value];
  • To filter the result set by multiple conditions, use the AND and OR keywords with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] = [value] [AND/OR] [col_name] = [value];
  • To filter the result set by a text string fuzzy match, use the LIKE keyword and the % wildcard operator with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] LIKE 'text_str%' OR [col_name] LIKE '%text_str' OR [col_name] LIKE '%text_str%';
  • To filter the result set with a single negative condition, use the NOT keyword with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE NOT [col_name] LIKE 'text_str%';
  • To filter the result set with a single condition within a range of numerical values, use the BETWEEN keyword in conjunction with the AND keyword with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] BETWEEN [min_val] AND [max_val];
  • To filter a result set by a list of values, use the IN keyword with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] IN([val1], [val2], [val3]);
  • To filter a result set by a null condition, use the IS NULL or the IS NOT NULL keywords with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] IS [NOT] NULL;

GROUP BY Clause

If you're able to insert data, you must be able to delete that same data. Let's learn how to do just that:

    How to Delete Data From a Table

Summary:

  • To delete all the data in a table, refer to the TRUNCATE statement in the previous tutorial.
  • To delete specific rows in a table, use the following syntax:
    DELETE FROM [table_name] WHERE [condition];
