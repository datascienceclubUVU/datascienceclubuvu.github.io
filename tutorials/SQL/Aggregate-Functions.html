SQL Tutorials: Aggregate Functions

DISCLAIMER: Before starting this tutorial, it's highly recommended that you download an RDBMS to follow along. We recommend using either PostgreSQL, MS SQL Server, or MySQL.

Isn't querying fun? Data Analysis is all about asking and answering questions using querying techniques such as SELECT statements. Let's expand our querying skills by learning how to work with numerical data. This lesson will cover the following learning objectives:

  • COUNT Function
  • SUM Function
  • AVERAGE/AVG Function
  • MIN/MAX Functions

COUNT Function

The COUNT Function can be used for both finding the number of items in a group or fidning the total number of rows in a table, as seen in the previous tutorial. Watch the following video to get a better understanding of this function:

    Intro to SQL Count Function

Summary:

  • The Result Set is the rows returned as a result of the query executed.
  • To retrieve all rows in a specified table, use the following syntax:
    SELECT *
    FROM [table_name];
  • To retrieve the total number of rows in a specified table, use the following syntax (We will go over the COUNT function in the next tutorial):
    SELECT COUNT(*)
    FROM [table_name];
  • To retrieve all the rows in a specified table but limit which columns are returned, use the following syntax:
    SELECT [col1], [col2], [col3], [col4]
    FROM [table_name];
  • To filter the rows returned in the result set, use the WHERE clause with the following syntax:
    SELECT *
    FROM [table_name]
    WHERE [condition(s)];
  • To order the result set in either ascending or descending order, use the ORDER BY clause with the following syntax:
    SELECT *
    FROM [table_name]
    WHERE [condition(s)]
    ORDER BY [col_name] [ASC/DESC];
  • When working with large result sets, it's a best practice to use the LIMIT Keyword to reduce the amount of memory your query uses:
    SELECT *
    FROM [table_name]
    WHERE [condition(s)]
    ORDER BY [col_name] [ASC/DESC]
    LIMIT [integer];

SUM Function

Although you have used the WHERE clause before, there's a lot more to it. Let's explore this clause in the video below:

    How to Filter with the WHERE Clause in SQL

Summary:

  • To filter the result set with a single condition, use the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] = [value];
  • To filter the result set by multiple conditions, use the AND and OR keywords with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] = [value] [AND/OR] [col_name] = [value];
  • To filter the result set by a text string fuzzy match, use the LIKE keyword and the % wildcard operator with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] LIKE 'text_str%' OR [col_name] LIKE '%text_str' OR [col_name] LIKE '%text_str%';
  • To filter the result set with a single negative condition, use the NOT keyword with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE NOT [col_name] LIKE 'text_str%';
  • To filter the result set with a single condition within a range of numerical values, use the BETWEEN keyword in conjunction with the AND keyword with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] BETWEEN [min_val] AND [max_val];
  • To filter a result set by a list of values, use the IN keyword with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] IN([val1], [val2], [val3]);
  • To filter a result set by a null condition, use the IS NULL or the IS NOT NULL keywords with the following syntax:
    SELECT [col1], [col2], [col3]
    FROM [table_name]
    WHERE [col_name] IS [NOT] NULL;

AVERAGE/AVG Function

Wow. That last section was a long one, yet an important one. Once you know how to fitler data, your querying skills go from nothing to something. Let's level up your querying skills by learning how to aggregate data using the GROUP BY clause:

    How to Aggregate Data in SQL
    Difference Between WHERE and HAVING Clauses

Summary:

  • The GROUP BY clause is used in conjunction with aggregate functions (e.g., COUNT(), AVERAGE/AVG(), SUM()) to gather identical data into groups.
  • To delete group rows in a table based on one or more columns, use the following syntax:
    SELECT [col1], [col2], COUNT(*)
    FROM [table_name]
    GROUP BY [col1], [col2];
  • The HAVING clause is used in conjunction with the GROUP BY clause and is sued to filter aggregated columns. This is used with the following syntax:
    SELECT [col1], [col2], COUNT(*)
    FROM [table_name]
    GROUP BY [col1], [col2]
    HAVING COUNT(*) >= 5;
  • The difference between the WHERE clause and the HAVING clause is that the HAVING clause can only handle conditions based on aggregated columns, whereas the WHERE clause handles conditions based on non-aggregated columns.
  • QUICK TIP: When dealing with a large number of columns, you can use column indexes when specifying columns in the GROUP BY clause rather than writing out the full column name. This is done with the following syntax:
    SELECT [col1], [col2], [col3], COUNT(*)
    FROM [table_name]
    GROUP BY 1, 2, 3;

MIN/MAX Functions

Once you've retrieved the correct columns, filtered the result set, and aggregated data, you'll want to organize it to make it more readable. The following video explains how to do this:

    How to Order Data in a Result Set

Summary:

  • To order a result set by a specific column in Ascending order (A-Z, Lowest-Highest), use the following syntax:
    SELECT *
    FROM [table_name]
    ORDER BY [col_name];
  • To order a result set by a specific column in Descending order (Z-A, Highest-Lowest), use the following syntax:
    SELECT *
    FROM [table_name]
    ORDER BY [col_name] DESC;
  • To order a result set by multiple columns in either ascending or descending order, use the following syntax. This will sort the result set by the first specified column, THEN by the second column.
    SELECT *
    FROM [table_name]
    ORDER BY [col_name] [DESC], [col_name] [DESC];
  • To order a result set by specific column(s) and restrict the number of rows returned, use the following syntax:
    SELECT *
    FROM [table_name]
    ORDER BY [col_name] [DESC], [col_name] [DESC]
    LIMIT [integer];
