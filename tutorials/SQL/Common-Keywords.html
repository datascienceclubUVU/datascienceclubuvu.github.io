SQL Tutorials: Common Keywords

DISCLAIMER: Before starting this tutorial, it's highly recommended that you download an RDBMS to follow along. We recommend using either PostgreSQL, MS SQL Server, or MySQL.

The most difficult part of querying is writing a query that other people can understand. This is where keywords come in. In SQL, Keywords are used to make your SQL queries more readable and user-friendly. There are tens of keywords in SQL, so we'll just cover the following learning objectives:

  • SELECT Statement Keywords
  • WHERE Clause Keywords
  • ORDER BY Clause Keywords

SELECT Statement Keywords

Writing clean SQL is what differentiates a Junior and Senior Data Professional. Since you have some experience writing basic SELECT statements, let's kick it up a notch by using the following keywords:

  • AS. This keyword is used to assign an alias to a newly created column. Although not required, it's a best practice to use this keyword to make it more obvious that the column is being temporarily renamed. To use this keyword, use the following syntax:
    SELECT [col_name] AS [new_name]
    FROM [table_name];
  • DISTINCT. This keyword is used to remove any duplicated data in the result set. One downside to this keyword is that it decreases performance, though if you have duplicate data, then it's better to have a slower query than an incorrect result set. To use this keyword, use the following syntax:
    SELECT DISTINCT [col_name], [col_name], [col_name]
    FROM [table_name];
  • TOP. This keyword is used by some RDBMSs to limit the number of rows in the result set. It's a best practice to use this keyword when working with large datasets. In some RDBMSs, the LIMIT keyword is used rather than this keyword. To use this keyword, use the following syntax:
    SELECT TOP [integer] [col_name], [col_name], [col_name]
    FROM [table_name];

WHERE Clause Keywords

In the previous tutorial, we learned how to filter data in a variety of ways using the WHERE clause. The purpose of this section of this tutorial is to summarize the keywords mentioned in the previous tutorial. Listed below are some common keywords used in conjunction with the WHERE clause:

  • NOT. This keyword is used to create a negative condition. This is useful when working with multiple logical conditions or complex logical conditions. To use this keyword, use the following syntax:
    SELECT [col_name], [col_name]
    FROM [table_name]
    WHERE NOT [condition];
  • AND. This keyword is used to filter the result set by more than one condition. To use this keyword, use the following syntax:
    SELECT [col_name], [col_name]
    FROM [table_name]
    WHERE [condition1] AND [condition2];
  • OR. Much like the AND keyword, this keyword is used to filter the result set by more than one condition. However, this is also used to create a more flexible filter on the result set. To use this keyword, use the following syntax:
    SELECT [col_name], [col_name]
    FROM [table_name]
    WHERE [condition1] OR [condition2];
  • BETWEEN AND. This keyword is used to filter a numeric column by specifying a range of values. To use this keyword, use the following syntax:
    SELECT [col_name], [col_name]
    FROM [table_name]
    WHERE [numeric_col] BETWEEN [min_value] AND [max_value];
  • LIKE. This keyword is used to filter a textual column by specifying a substring to be retrieved. This is used in conjunction with the % wildcard to specify whether the specified substring is at the beginning, middle, or end of the actual string. To use this keyword, assuming the desired substring is in the beginning, use the following syntax:
    SELECT [col_name], [col_name]
    FROM [table_name]
    WHERE [text_col] LIKE '[substr]%';
  • IN. This keyword is used to filter a result set by a list of desired values. These values can be either numeric values or string values, but never both. To use this keyword, use the following syntax:
    SELECT [col_name], [col_name]
    FROM [table_name]
    WHERE [col_name] IN([value1], [value2], [value3]);
  • IS NULL. This keyword is used to filter a result set by a row's NULL status. This is helpful when exploring datasets or identifying outliers for computations. To use this keyword, use the following syntax:
    SELECT [col_name], [col_name]
    FROM [table_name]
    WHERE [col_name] IS NULL;

ORDER BY Clause Keywords

You have already used the keywords described in this section, though it never hurts to review them. When sorting result sets, you have two options: Ascending order or Descending order. Let's take a look at how to best sort result sets below:

  • ASC. This keyword is used to order the result set in ascending order. Although this is the default sort order in every RDBMS, it's a best practice to specify the sort order for readability purposes. To use this keyword, use the following syntax:
    SELECT [col_name], [col_name]
    FROM [table_name]
    ORDER BY [col_name] ASC;
  • DESC. This keyword is used to order the result set in descending order. To use this keyword, use the following syntax:
    SELECT [col_name], [col_name]
    FROM [table_name]
    ORDER BY [col_name] DESC;
