SQL Tutorials: DDL

DISCLAIMER: Before starting this tutorial, it's highly recommended that you download an RDBMS to follow along. We recommend using either PostgreSQL, MS SQL Server, or MySQL.

Now that you have a clear understanding of the core concepts behind Relational Databases, let's learn some basic SQL. Data Definition Language (DDL) is a form of SQL used specifically for creating, altering, and dropping database objects. Listed below are the learning objectives covered in this lesson:

  • Creating Database Objects
  • Altering Database Objects
  • Dropping Database Objects
  • Truncating Database Tables
  • Constraints

Creating Database Objects

As mentioned in the previous tutorial, Relational Databases are comprised of four primary objects: Databases, Schemas, Tables, and Attributes. Watch the following videos to get a better understanding of these concepts:

    How to Create A Database
    How to Create A Schema
    How to Create A Table

Summary:

  • The CREATE DATABASE statement is used to create databases in an RDBMS. Once you have multiple databases created, it's helpful to use the optional IF NOT EXISTS keyword to check if the name has already been used.
  • The CREATE SCHEMA statement is used to create a schema within a database. These can be used to differentiate different business units and provide security for sensitive data. It's recommended to use this statement in conjunction with the USE DATABASE [db_name] statement to ensure you create the schema in the correct database.
  • The CREATE TABLE statement is used to define the name and structure of a table in a database. You must specify each column and its associated data type in the table. It's recommended to use this statement in conjunction with the USE DATABASE [db_name] statement to ensure you create the table in the correct database.

Altering Database Objects

When working with large amounts of data originating from various sources, you may need to make some changes to existing database objects. Watch the following videos to get a better understanding of these concepts:

    How to Alter Database Tables
    How to Rename Databases

Summary:

  • To add a column/attribute to a table, use the following syntax:
    ALTER TABLE [table_name] ADD [col_name] [data_type];
  • To drop a column/attribute from a table, use the following syntax:
    ALTER TABLE [table_name] DROP [col_name];
  • To change an attribute's data type, use the following syntax (differs between RDBMSs):
    ALTER TABLE [table_name] MODIFY [col_name] [new_data_type] [constraint];
  • To rename a database, use the following syntax (differs between RDBMSs):
    ALTER DATABASE [db_name] MODIFY name = [new_name];

Dropping Database Objects

As mentioned above, when working with large amounts of data, you may need to drop, or delete, existing database objects to make room for new and improved data. Watch the following videos to get a better understanding of this concept:

    How to Drop Table Columns
    How to Drop Database Tables
    How to Drop Database Schemas
    How to Drop A Database

Summary:

  • To drop a column in a table, use the following syntax (differs between RDBMSs):
    ALTER TABLE [table_name] DROP COLUMN [col_name];
  • To drop a table in a database, use the following syntax (differs between RDBMSs). This should be used in conjunction with the USE [db_name]; statement to ensure you drop the correct table.
    DROP TABLE [IF EXISTS] [table_name];
  • To drop a database schema, use the following syntax (differs between RDBMSs):
    DROP SCHEMA [IF EXISTS] [schema_name];
  • To drop a database, use the following syntax (differs between RDBMSs):
    DROP DATABASE [IF EXISTS] [db_name];

Truncating Database Tables

When you Truncate a database object, you remove all the existing data and keep the structure. This is useful for when you are testing out a new RDBMS or a new schema design. Watch the following video to get a better understanding of this concept:

    How to Truncate A Database Table

Summary:

  • Truncating involves removing all the data from a table to reconstruct the table's schema or restructure a schema without altering the data.
  • To truncate a table, use the following syntax (differs between RDBMSs):
    TRUNCATE TABLE [table_name];

Constraints

When dealing with large amounts of data, you'll want to place some limitations on the types of data or how the data is presented. These are known as Constraints and can be placed on the Schema, Table, or Column level. Watch the following video to get a better understanding of this concept:

    Creating Column-Level Constraints using the CREATE TABLE Statement
    Types of Column-Level Constraints
    Types of Table-Level Constraints

Summary:

  • To create a column-level constraint using the CREATE TABLE statement, use the following syntax (differs between RDBMSs):
    CREATE TABLE [table_name]([col_name] [data_type] CONSTRAINT [constraint_name] [constraint_type], etc.)
  • There are six primary column-level constraints: Primary Key, Foreign Key, Unique, Not Null, Check, and Default.

Exercise

Congratulations! You just completed the DDL Tutorial! We hope you learned a lot and feel more prepared to enter the workforce. To help test your knowledge, let's complete a simple exercise.

Instructions:

  1. Get access to an RDBMS system (e.g., PostgreSQL, MS Sql Server, MySQL)
  2. Create a Database named "Sales".
  3. In the 'Sales" Database, create two schemas named "Orders" and "Employees".
  4. Using the "Orders" schema, create the following four tables:
    Customers
        customer_id (INT) [PRIMARY KEY]
        first_name (VARCHAR(100)) [NOT NULL]
        last_name (VARCHAR(100)) [NOT NULL]
        street (VARCHAR(500))
        city (VARCHAR(100))
        state (CHAR(2))
        zip (CHAR(5))

    Orders
        order_id (INT) [PRIMARY KEY]
        order_date (DATE)
        customer_id (VARCHAR(100)) [FOREIGN KEY]

    Order_Items
        order_id (INT) [PRIMARY KEY/FOREIGN KEY]
        item_num (INT) [PRIMARY KEY]
        item_name (VARCHAR(250)) [NOT NULL]
        item_description (VARCHAR(250))

    Colors
        color_id (INT) [PRIMARY KEY]
        name (VARCHAR(50)) [NOT NULL]
        description (VARCHAR(100))
  5. Drop the "Colors" table.
  6. Alter the "Order_Items" table by adding the following column: item_price (FLOAT).
  7. Alter the "Customers" table by changing the data type of the "state" column to VARCHAR(50).
  8. Drop the "Employees" schema.
  9. Design an ERD to visualize the "Orders" schema.
