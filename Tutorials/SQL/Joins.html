BEGINNER SQL: JOINS & UNIONS



Introduction

SQL primarily deals with Relational Databases. The reason why these types of databases are named "relational" is because the objects can have relationships. How are objects related? That's what we'll be learning in this tutorial.

Definitions

Before we dive into how objects are related, let's go over some of the terminaology used in this tutorial:

  • Primary Key: This describes the column in a table that is used to uniqely identify a row.
       Example: UVIDs, Social Security Numbers, Driver License Numbers may all be Primary Keys.
  • Foreign Key: This is a column in a table the references the primary key of another table.
       Example: The UVID column in a table that contains information about courses students are taking.

Now that we are caught up on the terminology, let's look into how to build relationships between tables:

INNER JOIN

The INNER JOIN operation is the most common type of join. This is used to retrieve rows from two tables that have matching primary and foreign keys.

The inner join is unique because you don't need to use the "inner" keyword when creating the join. It's also worthy to note that if you select all columns from the table specified in the FROM clause, it will return all columns from both tables.

Example

   SELECT reportID, storeName, state, month, year, totalSales, foodSales, cleaningSales, clothingSales, stores.storeID*
   FROM salesReports AS reports
   JOIN stores ON reports.storeID = stores.storeID
   *if the two tables have a column that has the same name, you must specify which table to pull it from

Output


LEFT/RIGHT JOIN

The LEFT JOIN operation returns all rows from both tables but only the rows from the LEFT table in the join operation will have all values filled. If the keys don't match or don't exist the rows will return all NULL values. The RIGHT JOIN operation is the complete opposite of the LEFT JOIN. All the rows in the RIGHT table in the join operation will have all values filled.

Examples

   SELECT reportID, storeName, state, month, year, totalSales,
   foodSales, cleaningSales, clothingSales, stores.storeID
   FROM salesReports AS reports
   LEFT JOIN stores ON reports.storeID = stores.storeID

   SELECT reportID, storeName, state, month, year, totalSales,
   foodSales, cleaningSales, clothingSales, stores.storeID
   FROM salesReports AS reports
   LEFT JOIN stores ON reports.storeID = stores.storeID

FULL JOIN

The AVG (average) function finds the total of all the numerical values and divides it by the count of the rows included. This is used for comparing trends across groups.

Example

   SELECT state, AVG(sales) AS avgSales, year
   FROM dbo.sales
   GROUP BY state, year
   ORDER BY avgSales DESC;

Output


UNIONS

The MIN function finds the smallest value in either a numerical or categorical column. For a categorical column, it will return the string that begins with the letter closest to "A". This function is used to find the range of values within numerical columns or compare categorical values.

Example

   SELECT state, MIN(sales) AS minSales, year
   FROM dbo.sales
   GROUP BY state, year
   ORDER BY minSales DESC;

Output


MAX

The MAX function finds the largest value in either a numerical or categorical column. For a categorical column, it will return the string that begins with the letter closest to "Z". This function is used to find the range of values within numerical columns or compare categorical values.

Example

   SELECT state, MAX(sales) AS maxSales, year
   FROM dbo.sales
   GROUP BY state, year
   ORDER BY maxSales DESC;
