BEGINNER SQL: Data Manipulation Language (DML)



Introduction

As we discussed in the previous tutorial, databases are a collection of objects that can be created, altered, and dropped. The other subset of SQL is known as Data Manipulation Language (DML) and is primarily used for adding, updating, and deleting data from database objects.

DML Statements

DML offers a simple assortment of statements to add items to, update items in, and delete items from database objects. Listed below are the most common DML statements used when maintaining a Relational Database:

  • INSERT INTO [table name] VALUES ([values in order of table columns]): This allows you to load a single item into a database table.

  • INSERT INTO [table name] VALUES ([values in order of table columns]), ([second row of data])... : This allows you to load multiple items into a database table.

  • UPDATE [table name] SET [column name] = [value] (WHERE [condition] -- This is optional): This allows you to update column values in a database table.

  • DELETE FROM [table name] WHERE [condition]: This allows you to delete items from a database table based on a specific ondition. If no condition is provided, all values will be deleted.

Example

   USE sales; -- This specifies the database to use
   INSERT INTO managers.salesReps VALUES(123, 'Josh', 'Tyson', '7/2/2022', NULL);
   UPDATE VIEW totalSalesByRep SET firstName = 'Josh' WHERE firstName = 'Jsh';
   DELETE FROM salesReps WHERE hireDate < '01-01-2017';
